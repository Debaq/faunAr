# CONFIGURACIÓN DE CACHÉ - FaunAR v5

## ¿QUÉ SERVIDOR WEB USO?

### Opción 1: Verificar desde el navegador
1. Abre tu sitio web en Chrome/Firefox
2. Presiona F12 (abrir DevTools)
3. Ve a la pestaña "Network" o "Red"
4. Recarga la página (F5)
5. Haz clic en cualquier archivo (index.html)
6. Busca en los "Response Headers" la línea "Server:"
   - Si dice "Apache" → usas Apache
   - Si dice "nginx" → usas Nginx
   - Si dice "cloudflare" → usas Cloudflare (ver más abajo)

### Opción 2: Preguntar a tu proveedor de hosting
Contacta a tu proveedor (Hostinger, GoDaddy, etc.) y pregunta:
"¿Qué servidor web usa mi hosting?"


## CONFIGURACIÓN SEGÚN TU SERVIDOR

### SI USAS APACHE:
✅ Ya está listo! El archivo .htaccess se subirá automáticamente
   Solo asegúrate que el archivo .htaccess esté en la raíz del proyecto

### SI USAS NGINX:
1. Accede al panel de control de tu hosting
2. Busca la configuración de Nginx
3. Copia el contenido del archivo "nginx.conf" que está en este proyecto
4. Pégalo dentro del bloque "server { ... }" de tu configuración
5. Reinicia Nginx

### SI USAS CLOUDFLARE:
1. Entra a tu panel de Cloudflare
2. Ve a "Caching" → "Configuration"
3. Haz clic en "Purge Everything" (limpiar todo)
4. Configura las reglas de caché:
   - Browser Cache TTL: 1 hour
   - Development Mode: ON (por 3 horas mientras actualizas)

### SI NO TIENES ACCESO AL SERVIDOR:
✅ No te preocupes! Ya implementé dos métodos que funcionan sin configuración:

1. **Meta tags anti-caché** (en index.html y viewer.html)
   - Líneas 6-8: evitan que el navegador guarde caché de HTML

2. **Versionado ?v=5** (en todos los archivos JS y CSS)
   - Cuando cambies el número (v=5 → v=6), fuerza descarga nueva


## PARA FUTURAS ACTUALIZACIONES

Cuando hagan cambios importantes:

1. Abrir index.html y viewer.html
2. Buscar todos los "?v=5"
3. Cambiar a "?v=6" (siguiente número)
4. Guardar y subir

Ejemplo:
Antes: <script src="js/ar-engine.js?v=5"></script>
Después: <script src="js/ar-engine.js?v=6"></script>


## VERIFICAR QUE FUNCIONA

1. Abre el sitio en modo incógnito
2. Presiona F12 → Network
3. Recarga la página
4. Verifica que los archivos JS/CSS digan "?v=5" en la URL
5. Si ves archivos sin versión o con v=4, significa que hay caché


## AYUDA RÁPIDA

¿No funciona? Usuarios siguen viendo versión antigua:

1. Cambiar ?v=5 a ?v=6 en todos los archivos HTML
2. Si usas Cloudflare: Purge Everything
3. Pedir a usuarios: Ctrl+F5 (forzar recarga)
4. En móviles: Limpiar caché del navegador
